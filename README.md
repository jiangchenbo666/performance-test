# JSONPlaceholder 接口分布式性能测试
## 测试环境
- 服务器：阿里云 ECS 4核8G
- 压测工具：JMeter 5.6.3（Master-Slave 分布式架构）
- 测试接口：JSONPlaceholder 公开接口

## 核心测试结果（含拐点区间验证）
| 并发数 | 平均 TPS | 平均响应时间 | 错误率 | 核心结论 |
|--------|----------|--------------|--------|----------|
| 50     | ~290/s   | ~700 ms      | <0.05% | 资源富余，性能冗余大 |
| 200    | ~350/s   | ~750 ms      | <0.07% | 性能稳步提升，无瓶颈 |
| 400    | 388.1/s  | 845 ms       | 0.07%  | 逼近性能上限，接近拐点 |
| 450    | 385.1/s  | 850 ms       | 0.07%  | 达到性能拐点，TPS 停滞 |
| 500    | 391.1/s  | 839 ms       | 0.06%  | 进入性能平台期，无提升 |
| 550    | 386.0/s  | 849 ms       | 0.08%  | 性能平台期，无明显恶化 |
| 1500   | 388.8/s  | 849 ms       | 0.09%  | TPS 持平，长尾请求略增 |
| 2000   | 389.5/s  | 844 ms       | 0.09%  | 性能天花板，无提升 |

## 性能拐点分析（企业级结论）
1. **拐点区间**：400~450 并发为该接口的性能拐点；
2. **吞吐量上限**：接口最大处理能力约 390 TPS，拐点后 TPS 不再随并发数提升；
3. **稳定性**：拐点后（450~2000 并发）错误率始终 < 0.1%，平均响应时间 < 1s，接口表现可靠；
4. **瓶颈分析**：性能瓶颈为接口/网络，而非 4核8G 服务器资源（服务器 CPU 利用率仅 13%~70%）。

## 结果文件说明
- `result_*.jtl`：各并发数的原始压测数据（含 50/200/400/450/500/550/1500/2000 并发）；
- `report_*/`：各并发数的 HTML 可视化报告（可选）。

## 优化建议
- 生产环境建议将并发限流在 400 以内，避免进入性能平台期；
- 若需提升吞吐量，可从接口侧（缓存、异步化）或网络侧（CDN、多地域部署）优化。
